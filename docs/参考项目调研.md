## 综合借鉴结论（落到我们方案）

1. **元数据源选型被反复验证**：OpenLibrary 是社区最常用的免费数据源（booklogr / 多个 tracker 都用它），一期采用它做主源是稳妥选择；calibre/komf 的"多源回退 + 字段映射"值得照搬到我们的 `MetadataProvider`。
2. **数据模型对齐 jelu / mybibliotheca**：它们的"书(book) / 副本或文件(copy) / 用户读书状态(reading state) / 标签"分层与我们设计一致，说明方向正确；可去看它们的迁移脚本细化字段。
3. **录入体验对标 ReadingList(iOS) / mybibliotheca**：条码/ISBN 扫码 → 自动补全 → 用户确认，是被验证的高效录入路径，强化我们"拍照/扫码 + 确认"的交互。
4. **二期 Web 书架对标 calibre-web / komga / kavita**：封面墙、筛选、详情页、在线阅读、进度回写，信息架构可直接参考。
5. **OPDS 作为可选开放接口**：二期可考虑暴露 OPDS，让系统兼容市面阅读器 App，零成本扩大"前端"。
6. **差异化点（我们的独特价值）**：以上项目几乎都是"网页/App 表单录入"，**没有以 IM(飞书/微信/Telegram) + 多模态 Agent 作为入口**的——这正是我们方案最大的差异化与价值点，值得保留并打磨。
